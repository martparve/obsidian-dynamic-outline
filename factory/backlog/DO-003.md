Implement drag-and-drop reordering of outline headings. Dragging a heading in the outline moves the entire section (heading + all content until the next same-or-higher-level heading) in the source document. If dropped at a different heading level, all headings in the moved section adjust accordingly.

## Requirements

1. Make outline `li` elements draggable. Add `draggable="true"` and `data-heading-index` attributes in `outlineLiElement.ts`.

2. Add drag event handling in `outlineWindow.ts` using event delegation on the content container:
   - `dragstart`: store dragged heading index, add `.dragging` class
   - `dragover`: show drop indicator (accent-colored line above/below hovered item)
   - `drop`: perform the section move
   - `dragend`: clean up classes and state

3. Section range calculation: a section spans from the heading's line to the line before the next heading at same-or-higher level (or end of document).

4. Move logic:
   - Extract source section lines from the document
   - Calculate target level from the heading BELOW the drop point (if dropping at end, keep source level)
   - Compute level delta and adjust all `#` markers in the section (clamp 1-6, abort if out of range)
   - Remove source lines, adjust target offset, insert adjusted lines
   - Preserve scroll position

5. Disable dragging during search (`.is-searching` active).

6. Prevent invalid drops: onto self, within own section.

7. CSS: `.dragging` opacity, `.drop-indicator-above`/`.drop-indicator-below` accent line indicators.
