title: Reliable collapse icon clicks with draggable li
status: fixed
priority: high
stage: impl
files:

The collapse icon click is unreliable because the browser's drag detection starts on `mousedown` (before `dragstart`), interfering with the click. The `_onDragStart` prevention is too late.

Fix: add a `mousedown` handler with `event.stopPropagation()` on the collapse icon span. This prevents the mousedown from reaching the draggable `li`, so the browser never enters drag-preparation mode for icon clicks.

In `outlineLiElement.ts`, where the collapse icon is created (inside the `if (canCollapse)` block), add:

```typescript
iconSpan.addEventListener("mousedown", (event) => {
    event.stopPropagation();
});
```
