---
id: DO-011
title: Harden single-top-level detection against mixed heading levels
status: fixed
priority: high
stage: impl
files:
  - src/components/outlineWindow.ts
---

A document with many H2 sections and one stray H1 mid-document (e.g., "# 7. Prioritization" among many "## Section" headings) incorrectly triggers single-top-level mode. This hides collapse icons on all tab-level-1 headings and breaks collapse/expand entirely.

Root cause: `hasMultipleTopLevelHeadings` is computed from raw heading levels (`h.level`). When there's one H1 among many H2s, `minLevel=1`, `topLevelCount=1`, so single-top-level activates. But the H2s are semantically independent sections, not children of the stray H1.

Fix: in `update()`, add a check that single-top-level only activates when the lone min-level heading is the FIRST heading. A stray H1 mid-document should not trigger it.

Change this block:
```typescript
const minLevel: number = Math.min(...headings.map((h) => h.level));
const topLevelCount: number = headings.filter(
    (h) => h.level === minLevel
).length;
const hasMultipleTopLevelHeadings: boolean = topLevelCount > 1;
```

To:
```typescript
const minLevel: number = Math.min(...headings.map((h) => h.level));
const topLevelCount: number = headings.filter(
    (h) => h.level === minLevel
).length;
const hasMultipleTopLevelHeadings: boolean =
    topLevelCount > 1 || (topLevelCount === 1 && headings[0].level !== minLevel);
```

When there's exactly one heading at the minimum level but it's NOT the first heading, treat it as if there are multiple top-level headings (i.e., don't activate single-top-level mode).
