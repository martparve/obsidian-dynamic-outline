Add minimalistic "Collapse all" / "Expand all" icon buttons to the outline modal, placed as a top toolbar row mimicking default Obsidian collapse/expand UX in the file browser.

## Requirements

1. Add a toolbar div as the first child of the `#dynamic-outline` container (before the search container), containing two `clickable-icon` buttons:
   - "Collapse all" using `setIcon(el, "chevrons-down-up")` from obsidian
   - "Expand all" using `setIcon(el, "chevrons-up-down")` from obsidian

2. Toolbar visibility: only show when collapsing is possible (`isCollapsingPossibleGlobally` is true in the `update()` method). Hide with class `hidden` otherwise. Also hide during search (CSS: `#dynamic-outline.is-searching .dynamic-outline-toolbar { display: none; }`).

3. Collapse all logic:
   - Add `collapsed` class to all `li.has-children:not(.is-single-top-level)`
   - Compute effective root level: if container has `has-single-top-level` class, root = minLevel + 1; otherwise root = minLevel (where minLevel = min of all `data-level` attributes)
   - Add `hidden-by-collapse` class to all `li` with `data-level > rootLevel`

4. Expand all logic:
   - Remove `collapsed` and `hidden-by-collapse` classes from all `li` elements

5. CSS styling for the toolbar:
   - `display: flex; justify-content: center; gap: 2px; padding: 4px 15px 0;`
   - Hidden state: `display: none`
   - Icon buttons use Obsidian's native `clickable-icon` class
   - Icon SVG size: `width: 16px; height: 16px`

## Implementation details

- In `outlineWindow.ts`:
  - Import `setIcon` from `obsidian`
  - Add `private _toolbarEl!: HTMLDivElement;` field
  - Create toolbar in `_createElement()` method, before the search container
  - Add `_collapseAll()` and `_expandAll()` private methods
  - In `update()`, after computing `isCollapsingPossibleGlobally`, toggle toolbar visibility: `this._toolbarEl.classList.toggle("hidden", !isCollapsingPossibleGlobally);`

- In `styles.css`:
  - Add toolbar container styles
  - Add search-hiding rule
