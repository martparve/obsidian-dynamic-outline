Dragging a heading left/right should change its level without moving it vertically. Drag left = promote (e.g., H2 to H1), drag right = demote (e.g., H1 to H2).

## Implementation

Track the drag start X position. During `dragover`, compute horizontal delta. If the item is dropped back on itself (same vertical position) with significant horizontal offset, change the heading level instead of moving the section.

### Changes to `outlineWindow.ts`

1. Add a `_dragStartX` field (number) alongside `_draggedIndex`.

2. In `_onDragStart`: record `event.clientX` as `_dragStartX`.

3. In `_onDragOver`: when hovering over the same item being dragged (targetIndex === _draggedIndex), show a horizontal level-change indicator instead of drop-above/below. Compute `deltaX = event.clientX - _dragStartX`. Each 40px of horizontal movement = one level change. Show a visual cue: add class `level-change-left` or `level-change-right` to the dragged li based on direction.

4. In `_onDrop`: if dropping on the same item (targetIndex === _draggedIndex) and there's horizontal offset, compute level delta from `deltaX / 40` (rounded, clamped so result stays 1-6). Call a new `_changeSectionLevel(headingIndex, levelDelta)` method instead of `_moveSection`.

5. Add `_changeSectionLevel(headingIndex: number, levelDelta: number)` method:
   - Get the section range
   - Validate all headings in section stay within 1-6 after adjustment
   - Get editor lines, apply `_adjustHeadingLevels` to the section lines in-place
   - Preserve scroll position

6. In `_onDragEnd`: clear `_dragStartX`.

### Changes to `styles.css`

Add visual feedback for horizontal drag:
```css
#dynamic-outline li.level-change-left {
    box-shadow: -3px 0 0 0 var(--interactive-accent);
}
#dynamic-outline li.level-change-right {
    box-shadow: 3px 0 0 0 var(--interactive-accent);
}
```
